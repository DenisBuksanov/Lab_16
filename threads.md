## Задания 4 и 5: Отмена и очистка ресурсов
**Описание выполнения:**
В функции `zapusk_45()` реализован механизм принудительного завершения потока извне. Дочерний поток выполняет функцию `otmena()`, в которой запускается бесконечный цикл `while(1)`, каждую секунду выводящий строку "[Дочерний] Работаю...".
Главный (родительский) поток ждет 2 секунды с помощью `sleep(2)`, а затем вызывает `pthread_cancel(tid)` для отмены дочернего потока.
Чтобы при отмене поток корректно освободил занятые ресурсы, внутри него используется макрос `pthread_cleanup_push()`, который регистрирует специальную функцию очистки `och_otm()`. При получении сигнала на отмену поток прерывается, вызывает эту функцию очистки, и на экран выводится сообщение: `[Очистка] Поток отменен. Ресурсы очищены.`.

## Задание 6: Алгоритм Sleepsort
**Описание выполнения:**
Реализован нестандартный алгоритм сортировки массива чисел с использованием потоков, обладающий заявленной асимптотикой O(N).
Дан исходный массив чисел: `4, 2, 19, 5, 18`. В функции `zapusk_6()` для каждого числа массива создается отдельный поток. В качестве аргумента в поток передается само значение числа (через динамическое выделение памяти `malloc`, чтобы разные потоки не перезаписали общую память).
В функции потока `sleepsort()` каждый поток засыпает на количество секунд, равное переданному числу (`sleep(val)`). Поскольку потоки, получившие меньшие числа, просыпаются раньше, они первыми выводят свои числа на экран. Таким образом, числа естественным образом выводятся в консоль в отсортированном по возрастанию порядке по мере пробуждения потоков. Перед завершением работы каждый поток очищает выделенную память с помощью `free()`.
